# opfs-tools (文件系统 API) 项目介绍

文件系统是许多领域程序的基石，所有通用编程语言都会内置完备的文件系统 API。

Web 很长一段时间没有提供完善的访问文件系统的规范，使得需要高频读写文件、大文件处理软件在 Web 端都会受到一些限制，比如音视频剪辑、游戏、数据库等等。

之前在浏览器中实现视频裁剪 (opens new window)、截帧 (opens new window)等相关功能时，发现缺少基本的操作文件的 API，比如读写、移动、复制文件。

而 OPFS (opens new window)(Origin private file system) 已经发布很久了 (Chrome 86)，它提供了访问文件系统的低级 API，且需要在 Worker 中访问才能发挥最佳性能，日常使用起来很是麻烦，于是就有了基于 OPFS 封装简单易用 API 的想法。

以下两个项目就是笔者近期的成果

  • opfs-tools (opens new window)：在浏览器中运行的简单、高性能、完备的文件系统 API
  • opfs-tools-explorer (opens new window)：管理 Web 站点的 OPFS 文件，类似操作系统的 Explorer、Finder

点击体验 opfs-tools-explorer demo (opens new window)

后续会再写一篇文章详细介绍 Web 中与文件相关的 API 和 opfs-tools 实现原理。
